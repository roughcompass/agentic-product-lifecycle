Chapter 02 — The artifact gap

The problem isn't bad engineering. It's missing context.

Discovery happens in a meeting. Requirements live in a doc no one updates. Architecture decisions are tribal knowledge. Success criteria are vibes.

Each phase generates understanding. Almost none of it survives the handoff to the next phase.

Chapter 07 — Cumulative intelligence

Research compounds. Context connects.

Every discovery effort writes to a knowledge graph. Personas, problems, evidence, competitors, decisions, constraints. All connected by typed relationships.

The second discovery is smarter than the first. The tenth builds on everything before it. Nothing starts from scratch.

Chapter 08 — What becomes possible

Things that happen
when knowledge persists.

01

PRDs write themselves

The prd-drafter pulls personas, problems, and evidence from the graph. You refine, not start.

02

Updates auto-generate

The reporter reads graph diffs and git history. Status updates come from the system of record, not memory.

03

Validation has teeth

POCs have falsifiable hypotheses and measurable criteria. The validator tracks outcomes and recommends go/no-go.

04

Feedback closes the loop

Shipped features get reviewed against success criteria. Learnings feed back into the graph for the next cycle.

05

Stories tell themselves

Story-weaver turns graph entities into scroll-driven narratives. Executive updates become presentations.

06

Pluggable, not locked in

Bring your own tools via MCP servers. Linear, Jira, Amplitude, Zendesk. The agents adapt to what's available.

Closer

Stop managing products from memory. Start managing them from knowledge.
